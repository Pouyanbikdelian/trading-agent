A Multi-Agent Trading System

An automated trading stack for Interactive Brokers — a committee of AI analysts that debates daily, a simulated portfolio manager that trades weekly, and mechanical guards that protect capital around the clock.

1Architecture in one paragraph

Two decision systems share one data layer. A systematic cross-sectional momentum strategy (top-K selection, weekly rebalance) executes against an IBKR paper account through a hard-blocking risk manager — the only path to orders in the entire codebase. In parallel, an LLM agent ensemble consumes the same market, news and macro state, debates under an adversarial protocol, and manages an independent $1M simulated sleeve. The two are benchmarked against each other and SPY on a common equity curve. Promotion is evidence-gated: the agent sleeve earns real (paper) capital only if its out-of-sample track record — return, drawdown, and forecast calibration — justifies it. No exceptions, no narrative overrides.

si,t = Pi,t−g / Pi,t−L − 1,  hold top-K by cross-sectional rank the rules-based sleeve: classic momentum with lookback L and a skip gap g to sidestep short-term reversal — the benchmark the agents must beat.

2How a trading day flows

Quiet markets cost nothing — the LLM only runs when scheduled or when a tripwire fires.

3The committee — a decorrelated forecast ensemble

Seven specialist agents receive deliberately partitioned information sets — the quant never sees headlines, the scout never sees the macro dial — so their takes are structurally decorrelated rather than seven paraphrases of the loudest number. Every take must include a falsifiable prediction (instrument, direction, horizon, confidence ∈ [0.5, 0.95]) that enters a scorecard and is graded against realized prices.

The specialists

  • Quant — trusts numbers: momentum, vol surface, macro dial
  • Narrator — geopolitics, central banks, crowd psychology
  • Street — sell-side consensus vs. price action
  • Position Coach — the book only: entries, P&L, thesis drift
  • Risk Officer — what can hurt us this week
  • Trader — the tape, liquidity, this-week timing
  • Scout — hunts the next big sector from headlines + momentum; committed capital outranks commentary

Then the adversaries

The Challenger red-teams the consensus: crowding, base-rate violations, regime-specific failure modes, and the steelman of the opposite side — each objection paired with the evidence that would falsify it.

The Manager aggregates with weights driven by each agent's graded calibration (hit rate, Brier score) rather than rhetorical confidence, and rules: risk-on / neutral / risk-off, plus the single observable that would flip the ruling.

Digest → Telegram. Every prediction → the scorecard. Disagreement is measured, not smoothed over.

BSa = 1⁄N ∑n ( pn − on )2 per-agent Brier score over graded predictions — a proper scoring rule, so stated confidence p is rewarded only when it matches outcomes o.

4The agent PM — constrained portfolio construction, on probation

Weekly, the PM agent conditions on the full week of committee rulings (posture trajectory, not a snapshot), per-agent calibration statistics, its own live performance vs benchmark, and fresh context — then emits target weights over a whitelisted universe (liquid ETFs + large-cap index constituents). Constraints are enforced post-hoc in code, where no prompt can negotiate: long-only, gross ≤ 1.0, 25% per ETF, 10% per single name, 40% per correlated cluster (semis + mega-cap tech treated as one risk factor; excess weight is cut to cash, never redistributed — concentration is a choice, diversification is not a consolation prize). Off-whitelist tickers are dropped silently: hallucination-proof by construction. Fills at the close with a 10 bps cost model; daily mark-to-market against SPY produces the public equity race.

w ∈ ℝn≥0,  ∑i wi ≤ 1,  wi ≤ wmaxtype(i),  ∑i ∈ C wi ≤ 0.40  ∀ clusters C the PM's feasible set — long-only, unlevered, per-name caps by instrument type, and a hard correlated-cluster constraint. Enforced in code after inference; infeasible weight is projected to cash.

5Protection — three independent layers

① Sentinel (tiered escalation)

Deterministic tripwires every 15 min (index shock, vol spike, position drawdown) cost nothing and never hallucinate. Only a tripped wire invokes inference: one agent classifies idiosyncratic vs systemic. Systemic → alert + auto-convened committee within minutes. Debounced to one escalation per shock. Detection is code; judgment is the model.

② Volatility-scaled trailing stops

Stop distance = k× each name's realized daily move, clamped to a floor (out of stop-hunt range) and a cap. High-water-mark logic means strength is never sold — only confirmed breakdowns from the peak. Exits route through the same audited, halt-aware command pipeline as a manual close. Operator-pinned positions are exempt by contract.

③ Profit ratchet (portfolio)

Arms once the portfolio is meaningfully up; warns loudly after giving back a set share of the peak gain. "Secure what you made" — the decision stays human.

How a trailing stop rides a winner

The stop only rises. A +140% runner is never sold on the way up — only on a real breakdown from its peak, keeping most of the gain.

Si,t = Mi,t · (1 − δi),  δi = clip( k σ̂i, δmin, δmax ) Mi,t = running high-water mark; σ̂i = realized mean absolute daily move. Monotone in M: the stop can only ratchet upward.

The learning loop — proper scoring, gated memory

Predictions are graded nightly against realized prices into per-agent calibration (hit rate, Brier score) — a proper scoring rule, so overconfidence is penalized, and the Manager's aggregation weights follow the evidence. Weekly, a Historian agent distills at most two candidate lessons from the journal; a lesson enters the agents' context only after +3 net supporting weeks of evidence, and is retired (never deleted) when contradicted. The LLM proposes; accumulated evidence disposes.

6Design invariants (non-negotiable)

  • Single order gateway. Strategies and agents emit signals (target weights); only the hard-blocking risk manager constructs orders. There is no second path, by architecture rather than convention.
  • Separation of inference and execution. LLM output is advisory state — journaled, graded, never directly executable. Constraints live in code, where they cannot be prompted away.
  • Paper until proven. Live trading is double-locked behind two explicit flags and never enabled by automation. Promotion follows the staging pipeline: simulation → paper sleeve → sized-down live.
  • Full auditability. Every take, objection, ruling, trade and lesson is journaled with its evidence — the system's epistemic state is inspectable at any time.

Tech stack

  • Python — pandas / NumPy, pydantic, a custom ~200-line vectorized backtester
  • Interactive Brokers via ib-async; paper gateway in Docker
  • Claude API for the agent layer — JSON-strict, provider-agnostic client
  • Free data only: yfinance, Google News RSS, FRED — total data cost $0
  • One small VPS (2 GB), docker compose, Telegram for ops, self-hosted dashboard
  • Tests on every change; hermetic agent tests with injected LLMs

The road to production

Phase 1 — observation: the agent PM trades simulated capital; judged on alpha vs SPY and the rules-based sleeve, max drawdown, and forecast calibration — not on narrative quality.
Phase 2 — paper sleeve: if the evidence clears the bar, the PM emits signals through the production risk manager at a minority capital allocation.
Phase 3 — live: only after ≥30 clean paper days, sized down, behind the double lock. LLM strategies cannot be backtested honestly, so the burden of proof sits entirely on forward performance.
